# bgrok iOS Client — Setup & Compilation Guide

This folder contains the native Swift/SwiftUI sources for the **bgrok** remote desktop iOS client. Follow these instructions to compile the project in Xcode and deploy it to your physical iPhone.

---

## Prerequisites
* A Mac computer running macOS.
* **Xcode** (Version 14+ recommended) installed on the Mac.
* A physical iPhone running iOS 15+.
* A USB to Lightning/USB-C cable to connect your iPhone to the Mac.
* Both your iPhone and Windows laptop connected to the **same Wi-Fi network**.

---

## Step 1: Create a new Xcode Project
1. Open Xcode on your Mac and select **Create a new Xcode project**.
2. Select **iOS** -> **App**, then click **Next**.
3. Fill in the project details:
   - **Product Name**: `bgrok`
   - **Organization Identifier**: e.g., `com.yourname`
   - **Interface**: `SwiftUI`
   - **Language**: `Swift`
4. Click **Next** and save the project folder on your Mac.

---

## Step 2: Add WebRTC Dependency via Swift Package Manager (SPM)
We use the WebRTC iOS SDK package to negotiate signaling streams.
1. In Xcode, select your project name in the left navigator pane.
2. Select the project target under **Project** (not Targets), then select the **Package Dependencies** tab.
3. Click the **+** (Add) button at the bottom of the list.
4. In the search box, paste the following Git URL:
   ```
   https://github.com/alexpiezo/WebRTC.git
   ```
   *(Note: This is a verified SPM wrapper around Google's official WebRTC framework).*
5. For Dependency Rule, select **Up to Next Major Version** (default version `1.1.0` or similar will resolve successfully).
6. Click **Add Package** and select **WebRTC** to add it to the `bgrok` target.

## Step 4: Configure Local Network Permission (Info.plist)
Since iOS 14, apps that communicate with local servers must declare Local Network access.
1. Select the `bgrok` target, go to **Info** tab.
2. Add the following keys:
   - **Privacy - Local Network Usage Description** (`NSLocalNetworkUsageDescription`): Set this string to `bgrok needs access to the local network to find and connect to your computer agent.`
   - **Bonjour services** (`NSBonjourServices`): Add a new item `_http._tcp` to allow HTTP signaling handshakes.
   - **App Transport Security Settings** (`NSAppTransportSecurity`): 
     - Add dictionary key: `Allow Arbitrary Loads` -> Set value to `YES` (This allows your phone to connect to local development agents using non-HTTPS `http://` IP addresses).

---

## Step 5: Run on Physical iPhone
1. Plug your iPhone into your Mac using the USB cable.
2. In Xcode, click the device selector at the top bar and select **your physical iPhone** as the target.
3. If this is your first time deploying:
   - Go to Project Settings -> **Signing & Capabilities** -> Select your **Team** (you can use a free personal Apple ID account).
4. Hit the **Run** button (Play icon) in the top-left of Xcode.
5. Xcode will build and install `bgrok` on your iPhone.
6. *If you see an "Untrusted Developer" prompt on your iPhone*:
   - On the iPhone, go to **Settings** -> **General** -> **VPN & Device Management**.
   - Tap your developer profile under Developer App, and select **Trust**.

---

## Step 6: Connect and Verify
1. Make sure the Windows Agent is running on your laptop:
   ```powershell
   python -m agent.main --port 8080
   ```
2. Get your laptop's local IP address by running `ipconfig` on the laptop (e.g., `192.168.1.15`).
3. Open **bgrok** on your iPhone.
4. Input the Agent LAN address: `http://<your-laptop-ip>:8080`.
5. Tap **Connect Session**.
6. Verify:
   - Your Windows desktop screen displays live on your phone screen.
   - Swiping on the Trackpad area moves your mouse cursor relative to the screen.
   - Double-tap or Long-press triggers a right click.
   - Modifier keys (Ctrl, Alt) turn purple when selected, letting you execute shortcut combos.
   - Tapping the Keyboard icon pops up your native keyboard, allowing typing into active Windows inputs.
